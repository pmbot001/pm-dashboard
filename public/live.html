直播模組的一級入口（/live），依後台設定走標準模式或純直播模式
PM 提醒：廣場有兩種模式：標準模式（最多 5 個 Tab）和純直播模式（pushType=2，只有直播+主播 2 個 Tab）。寫需求前先確認是哪種模式，因為 UI 殼不一樣。
廣場直播列表（全部 Tab）

規則Tab 結構 程式層

標準模式最多 5 個 Tab：直播（永遠）、排行榜（後台開關）、主播（永遠）、節目表（永遠）、影片專區（後台開關）。純直播模式只有直播 + 主播。

排行榜和影片專區 Tab 受後台開關控制，PM 測試前要先確認後台設定。

規則登入攔截 程式層

主播、排行榜 Tab 需要登入。未登入點擊會觸發登入 Sheet，不切換 Tab。排行榜有額外的遊客模式例外。

規則深連結 + Tab 記憶 程式層

URL 帶 ?active=ranking 可自動切到對應 Tab。如果目標 Tab 是 disabled，fallback 到直播 Tab。登出後也會自動切回直播 Tab。

規則自動播放 程式層

進入時 video.js 初始化 HLS 串流（autoplay + muted）。瀏覽器政策：必須先靜音才能自動播放。偵測到互動後自動解除靜音。

規則串流重試 程式層

串流播放失敗：首次立即重試，之後每 3 秒重試。canplaythrough 觸發後停止重試。

規則主播下播 / 轉移 程式層

主播下播（WS 推送）顯示 AnchorOfflineDialog + 推薦其他直播間。玩家被轉移顯示 PlayerTransferSheet。

UI暗色主題 程式層

進入時強制切換暗色主題，離開時還原。改主題系統要注意這個強制覆蓋。

🟡 模組耦合

改 A 模組要同時回測 B 模組的交叉依賴

Y-1 直播間進房依賴賽事詳情模組
描述

所有直播間的 WS 進房邏輯（useEnterRoomInit）定義在 pages/match-detail/composables/ 下，不在 live 模組內。

影響

改 useEnterRoomInit 的任何邏輯 → 影響所有直播間類型 + 賽事詳情頁 + 開房對賭 + 存款直播。

來源

pages/match-detail/composables/useEnterRoomInit.ts（7 處引用）

Y-2 liveStore 被 12+ 個外部模組引用
描述

stores/live/index.ts 被首頁彈窗、登入流程、充值、系統通知、購物車、goto 工具等模組引用。

影響

改 liveStore 的 getter/action 簽名 → 可能影響 12 個以上外部引用。

Y-3 開房對賭跨模組依賴
描述

open-room 引用 match-detail 的型別（SportId）、使用 useEnterRoomInit、引用 useChatRoom 的常數。

影響

改體育賽事的型別定義或進房初始化 → 要回測開房對賭。

Y-4 聊天室場館分派邏輯全局共用
描述

composables/useChatRoom/ 依場館分派 useFb/useOb/useIm/useImEsport，被直播間和開房對賭共用。

影響

改某場館的聊天室實作 → 要測所有使用該場館的直播間類型 + 開房對賭。

Y-5 主播頁爆料依賴八卦模組
描述

主播主頁的爆料 Tab 使用 PredictList 元件，底層引用 gossiping 模組。

影響

改爆料模組 → 要回測主播主頁的爆料 Tab。

Y-6 開房入口嵌在全站底部導覽列
描述

OpenRoomEntrance 元件嵌在 MainTabbar（4 個皮膚版本）+ TabBarTipOpenRoom。

影響

改開房入口的 UI 或路由 → 要測 4 個皮膚版本的底部導覽列。

Y-7 任務系統耦合
描述

直播列表和開房列表都引用 useFindPageMission + @luxon/missions，用於新手引導和任務進度。

影響

改任務系統的條件類型 → 要回測直播列表和開房的任務引導。

Y-8 充值導流帶直播間資訊
描述

useRecharge 在直播間內充值時會帶 activeLiveId、matchId、roomId 做打點。

影響

改充值流程或 liveStore 的 activeLiveData 結構 → 要測直播間內的充值路徑。
