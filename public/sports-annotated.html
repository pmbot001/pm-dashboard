2 首頁框架
Header、Banner、場館切換列 — 整個體育首頁的外殼
體育首頁（預設展開）
滑動後收合狀態

UIHeader Bar 程式層

永遠貼在頂部的那條灰色列。左到右：側欄開關、Logo、儲值按鈕、餘額、客服 Icon。

  • 餘額會自動更新，手動點旁邊也能刷新（但有冷卻時間，不讓玩家狂點）
  • 餘額本身是從錢包系統拉的，體育動不了；要改錢包顯示規則找錢包 PM
Header 的文字 / 圖示 / 排列順序都是寫死的。想改版面要發版。

規則往下滑會收合 程式層

玩家往下滑超過一段距離後，Banner 消失、Header 變小，讓賽事列表盡量填滿畫面。

收合的觸發距離是寫死的。想改收合時機要發版。

UI場館切換列 SportBar 程式層

橢圓型切換列：「娛樂 / 體育」切換 + 杯賽入口。杯賽模式下顏色會整個切換成白色系。

左右順序、顯示文字、顏色切換規則都是寫死的。

UIBanner 輪播 設定層

自動輪播的活動橫幅。運營在後台設定要上哪幾張、上架 / 下架時間、點了連去哪。

  • 設定了起迄時間的，過期會自動消失
  • 沒有圖片檔的會被過濾掉
上 / 下 Banner 是運營日常操作，不用發版。但圖片尺寸 / 排版 / 動畫是寫死的。

專業版視圖

UI球種快捷列 設定層

頂部的足 / 籃 / 網 / 電競… tab，會顯示每個球種有幾場賽事。點一下切換球種就會重新拉列表。

哪些球種顯示、是否開放，取決於場館那邊給了什麼。我們能做的是在後台白名單決定要不要把那個球種露出來。

規則時段切換 程式層

時段有：滾球、今日、早盤、串關、冠軍。

  • 沒賽事的時段會被隱藏
  • 預設優先顯示順序：滾球 > 今日 > 早盤 > 串關 > 冠軍
  • 早盤 / 串關會多出一列日期選擇器
時段的名稱、順序、「自動挑第一個有賽事的時段」這規則都是寫死的。

UI賽事卡片 改不了

每張卡顯示：主客隊名 + 隊徽 + 比分 + 時間 + 最多 3 個可點的賠率按鈕。

隊名、隊徽、比分、時間、紅黃牌、角球 全都是場館下發的，動不了。要改要去跟場館反映（幾乎不會成功）。

UI簡易盤 程式層

卡片上直接顯示的那幾個快速下注盤口（通常是讓球 / 大小 / 勝負）。每個場館能放哪幾個盤口規則不一樣。

每個場館的簡易盤組合是前端寫死的，要動要發版，而且要場館一個一個確認。

UI新手版 / 專家版切換 設定層

專家版會顯示更多盤口與資訊，新手版簡化。運營在後台可以配置每個皮膚的預設版型。

時段說明運營 / PM 要知道
滾球（LIVE）進行中的賽事比分即時更新、最熱門的時段
今日今天要開打的—
早盤未來幾天的預選賽事會多出日期篩選
串關可以串在一起下注的賽事會多出日期篩選
冠軍整個賽季冠軍歸屬下注是獨立列表，跟上面四個不通用
關注玩家自己收藏的賽事個人資料，不能跨裝置同步
4 賽事詳情
點進一場賽事後的完整資訊頁
PM 警報：賽事詳情的底層頁面殼是「共用殼」，被開房、充值直播房、爆料、真人娛樂、聊天室等多個模組共用。任何「改賽事詳情頁佈局」的提議都是大需求，不是小改動。（細節看 L2 narrative §1.5）

賽事詳情 — 影片 / 動畫

UI頂部比分板 改不了

主客隊名、Logo、即時比分、半場比分、紅黃牌、角球。上方可以切換：影片 / 動畫 / 比分板。

全部都是場館給的資料，動不了。包括「這場顯示不顯示紅黃牌」也是場館決定。

UI下方分頁 程式層

聊天 / 包賠 / 下單 / 競猜 / 賽況 / 爆料，預設停在「下單」。

分頁的數量、順序、哪個預設、名稱 全是前端寫死的。想改分頁要發版。

UI全盤口列表 改不了

分類：熱門 / 讓球 / 大小 / 勝平負 / 比分 / 單雙。盤口本身與賠率都是場館給的。

盤口「分組名稱」我們可以改（翻譯層），但「這個盤口屬於哪一類」場館決定。

UI影片播放 改不了

比賽直播影片，來源是場館給的串流網址。支援子母畫面 (PiP) 邊看邊下注。

影片能不能看、有沒有播、畫質多少，全是場館決定。業主抱怨「影片不清楚」要回絕。

UI競猜 / 主播 PK 盤口 程式層

⚠️ 跟場館盤口是兩套。這是我們主播自己推的 PK 題目，跟真實下注沒關係。

業主或業務跟你說「競猜盤的贏率太高」時，要分清楚他講的是場館真實盤口還是主播 PK 競猜——兩件事。

UI晒單 / 排行榜 / 推單 / 邀請 / 禮物 設定層

直播房旁邊的各種互動按鈕。每個房間可以獨立決定要不要開。

這些是運營每個房間獨立配的，不是全域開關。

UI包賠 / 活動進度條 / 直播任務 設定層

首存包賠彈窗、Rally 活動進度、直播任務進度。都是其他模組注入進來的。

想動這些東西要找對應模組的 PM（活動 PM / 任務 PM），體育不管。
5 投注購物車
玩家點下賠率到送出注單的那個彈出框
PM 警報：購物車是體育所有動線的匯合點——它同時承接「首頁簡易盤直接下注、推單、跟單、追注、首存包賠預設」五種來源。業主說「購物車很快啦、小改而已」的時候一律警覺，預設為大需求，工時 +30%。（細節看 L2 narrative §1.4）
單項投注
投注成功

規則單注流程

點賠率 → 購物車展開 → 輸入金額 → 確認 → 送出 → 顯示「等待結果」→ 成功 / 失敗彈窗。

UI購物車畫面 程式層

從上到下：場館名稱 + 餘額、選中的盤口 + @賠率、金額輸入框、可贏金額提示、送出按鈕。

版面、欄位順序、文案全部寫死。

規則下注金額上下限 設定層

每筆下注的最低 / 最高金額是場館 + VIP 等級雙軸決定的，後端回傳。快捷金額按鈕（100 / 500 / 1000…）也是後台配的。

業主想「調整最低下注額」要去後台改 VIP × 場館配置，不用發版。但修改公式（例：加一個新的計算維度）要發版。

規則賠率即時變動 改不了

玩家在購物車裡猶豫的時候，後端會推最新賠率過來。賠率變了會提示，玩家可以接受新賠率、拒絕、或設定「自動接受更好的賠率」。

賠率本身動不了（場館給的）。但「變動後的處理規則」我們可以改（接受 / 拒絕 / 彈窗）——那是前端寫死的。

規則重複下注保護 程式層

同一盤口 + 同樣金額 + 短時間內重複點 → 彈窗確認「你是不是按到兩次？」玩家可以勾「不再提示」。

冷卻時間、觸發條件都是寫死的。

規則餘額不足 程式層

金額 > 餘額 → 彈窗提示「餘額不足」，按鈕直接導去充值頁。

測試帳號會顯示「此為測試帳號，無法操作」而不是導去充值。

規則五種下注來源 程式層

購物車可以從這五個地方打開：

  • 首頁簡易盤直接點賠率
  • 推單（主播推的單被玩家點進去）
  • 跟單（複製其他玩家的注單）
  • 追注（上一注輸了追加）
  • 首存包賠（預先幫玩家擺好）
動購物車的需求 每條來源的 flow 都要跑一次。別只驗「從賽事列表點進去」這一條。
6 串關 / 預約下注 / 提前結算
三個進階玩法，場館支援度完全不對稱
PM 警報：這三個功能各場館支援度差很多。任何「全體育生效」的需求，先問「所有場館都包含在內嗎？」——通常答案是「只有 fb / 只有幾個場館」。（細節看 L2 narrative §1.3）
串關投注
預約下注

規則串關 程式層

選多場賠率 → 加入串關購物車 → 選串關方式（3 串 1、自由串等）→ 送單。統一通過或統一失敗，不會單場結算。

各場館的「串關方式名稱」不同（例：SABA 叫法跟 FB 叫法不一樣），不能以為玩家看到的是同一套。

規則預約下注 設定層

盤口還沒開的時候先預約一個目標賠率，等開盤後如果賠率達到就自動下注。

⚠️ 只有 FB 場館支援。其他場館都沒有。業主問「預約下注怎麼不能用」先確認他在哪個場館。

規則提前結算 設定層

賽事還沒結束就把注單「提早賣掉」，鎖住部分獲利 / 減少損失。

  • FB：支援部分結算、預約結算、最多結 5 次
  • OB / SABA：只能全額結，一次結清
  • IM-Esport：不支援
能不能提前結算、結算的金額 / 比例全都是場館決定的，前端只顯示。業主想「調整提前結算比例」要回絕——動不了。
7 冠軍玩法
對整個賽季的冠軍歸屬下注
冠軍賽事列表

規則冠軍流程 程式層

切到「冠軍」時段 → 先選聯賽 → 看各隊伍奪冠賠率 → 選一隊下注。

冠軍玩法是獨立列表，跟一般賽事列表完全不通用（不會混在一起顯示）。

規則FB 冠軍的特殊行為 改不了

FB 冠軍下注後不會加入「關注列表」，這是 FB 場館的特殊行為。其他場館正常會加進去。

這是歷史事故定下來的（Jira GOR-879），不要試著「修正」它。
8 場館系統
我們不是做市商，是「多場館賽事聚合器」
體育系統的本質：我們不自己出賽事、不自己定賠率、不自己做直播。所有這些都是向 8 個場館（FB / IM / OB / SABA / YOO / IM-Esport / LiveCasino / WS168）拿的。我們是聚合入口。
這決定了你的權力邊界：賠率、賽事、比分、影片、直播，全部動不了。業主想調這些的需求要當場拒絕。（L2 narrative §1.1 / §1.3）
側邊欄 — 場館切換
場館維護中

規則場館切換 程式層

切換場館時，會清空原本的球種、時段選擇，重新拉新場館的資料。

UI維護中畫面 設定層

場館維護中時，整個賽事列表區會被「維護中」頁面取代。維護狀態由運營在後台切換，或場館那邊通知。

規則場館不對稱 改不了

8 個場館不是對等的：支援的功能、玩法、時段、盤口類型各不相同。

任何「全體育生效」的需求都要先問 RD：每個場館都包含在內嗎？
場館玩家印象特殊行為
FB功能最全唯一支援預約下注、提前結算最完整、冠軍不加關注
OB電競強電競玩法特殊、部分玩法要整數投注
SABA標準玩法名稱跟別家不同
IM（自營）自營盤簡易盤規則是自己定的
YOO / WS168 / LiveCasino次要場館功能支援度最少，各種新功能通常不包含
IM-Esport電競專場不支援提前結算
9 搜尋與篩選
賽事搜尋、聯賽篩選、早盤日期選擇
賽事搜尋
篩選聯賽

規則賽事搜尋 程式層

輸入文字匹配「賽事名」或「球隊名」。最近搜過的字會留在本機，換裝置不同步。

業主問「搜尋紀錄為什麼換手機就沒了」要直接回絕——不值得做同步。

規則聯賽篩選 程式層

可多選聯賽。選了之後列表只顯示該些聯賽的賽事。

10 新手引導
首次進體育的玩家會看到的三步教學
教學 — 存款位置
教學 — 切換場館

規則三步引導 程式層

教學步驟：① 告訴你儲值按鈕在哪 → ② 告訴你場館切換在哪 → ③ 告訴你怎麼點賠率。完成後不再出現。

想改教學內容 / 順序 / 指向要發版。

規則特定皮膚的每日提示 程式層

有些皮膚會每天首次進場時顯示「場館切換」提示。

「哪些皮膚啟用」是寫死的。
11 即時更新
賠率 / 比分如何自動重新整理

規則自動更新頻率 程式層

登入狀態下，賠率與賽事列表每幾秒自動更新一次。未登入狀態頻率較低。

頻率是前端寫死的。業主問「為什麼我看到的賠率跟其他人的不一樣？」先確認他是不是未登入狀態。

規則切換時取消舊請求 程式層

玩家切場館 / 切球種 / 切時段太快時，前端會把還在跑的上一個請求取消掉，避免畫面跳來跳去。

規則頁面背景時停止更新 程式層

玩家切到其他 tab 或把 app 切到背景時，自動更新會暫停；回到前景時自動恢復。

這是省流量的設計。業主問「為什麼我打開 app 後要等一下賠率才會動」要解釋這是正常行為。
