15 跨服務限制

技術限制

客服 iframe 跨域 postMessage 通訊
描述
官方客服透過 postMessage 通訊（7 種 action：jump/close/view/queue/tips/move/keyboard），跨域環境下偶爾 message 遺失。
影響
測試域名未在白名單內會導致客服功能異常。
PM 對策
新環境上線前確認客服 iframe domain 已加入白名單。
爆料詳情雙重請求設計
描述
detail（盤口+metadata）和 content（HTML 富文本）分開請求，透過 predictId 關聯。
影響
首次載入稍慢，但降低主請求回應大小。content 回傳 ArticleContent 陣列含 HTML + imageMobile + imagePc。
PM 對策
有意設計，非 bug。
客服 iframe 全局持久不卸載
描述
掛在 app layout 層，不隨頁面切換卸載。全螢幕橫屏時寬度改為 50vw 靠右對齊；showService=false（進入三方遊戲）時 topOffset=2000px 全部隱藏。
影響
常駐佔用記憶體；橫屏遊戲中無法使用客服。
PM 對策
已知行為。
iOS 鍵盤彈出處理
描述
全版模式 + iOS 時，客服 iframe 發送 keyboard action。彈出延遲 150ms 設定 keyboardVisible=true，收起立即設 false。iframe 高度調整為 visualViewport.height。
影響
Android 無此問題。iOS Safari 虛擬鍵盤影響 viewport 高度。
PM 對策
測試 iOS 裝置客服輸入時畫面是否正常。
盤口動態載入
描述
爆料詳情使用 import.meta.glob('components/marketId/*.vue') 根據 layoutId 動態載入盤口組件。
影響
新增盤口類型需新增對應 Vue 組件檔案。
PM 對策
確認新盤口類型的 layoutId 已有對應組件。

模組耦合

爆料入口散布多個模組
描述
爆料入口在首頁、直播間、賽事詳情（isFromPredictDetail）、開房（fromOpenRoom）等，各入口的訪客檢查未統一。
影響
部分入口可能遺漏 gamePredictVisitorDisplayMode 檢查。
PM 對策
新增爆料入口需確認訪客攔截。返回導航各來源邏輯不同。
跟單投注整合體育購物車
描述
爆料跟單透過 formatCartData 格式化後加入體育購物車。推薦盤口帶 extInfo（pageFrom/comboType/idType/liveRoomID/predictId/pushOrderId/streamerID）。
影響
購物車結構改動會影響爆料跟單。setBetSource(predict) 標記來源。
PM 對策
體育購物車重構需同步測試爆料跟單。
消息中心未讀數由 popupStore 驅動
描述
公告未讀數從 popupStore.getNotReadNoticeCount 同步到 Tab badge。
影響
popupStore 初始化延遲會導致 badge 數字暫時為 0。
PM 對策
已知行為。

設定層限制

客服 VIP 線道分流由後台設定
描述
getCsLineConfig 回傳的線道含 vipBegin~vipEnd，前端 _btnFilter 依此過濾。specialLine 和 normalLine 各自有 enable 開關。
影響
VIP 升降級後線道切換不即時（快取值）。
PM 對策
VIP 相關客服問題需後台調整。
gamePredictEnabled 控制爆料開關
描述
後台關閉時，爆料首頁 onMounted 直接跳轉 /live。
影響
整個爆料模組一鍵關閉。
PM 對策
關閉前確認不影響活躍用戶。
gamePredictVisitorDisplayMode 訪客模式
描述
mode=1 允許訪客瀏覽列表和文章，但跟單/按讚仍需登入。其他值在 CardList.handleClick 階段直接攔截。
影響
部分入口未統一檢查此開關。
PM 對策
確認所有爆料入口的訪客邏輯。

業務限制

落地頁新舊用戶判斷依賴打點記錄
描述
新/舊用戶判斷是否曾下載過 APP，依打點記錄。
影響
清除瀏覽器資料後可能誤判為新用戶。
PM 對策
打點記錄不可跨瀏覽器/裝置。
URL 簽名用 MD5 加密
描述
客服 URL 的 sign = MD5(uid+agentid+mac+uname+avatar+theme+kefuSign)。timestamp 首次開啟生成，close 時重置。幣商跳轉時 imDealer 參數 spread 進 params。
影響
kefuSign 是 runtime config 環境變數，不同環境值不同。
PM 對策
新環境需確認 kefuSign 已設定。
客服自動恢復 10 分鐘逾時
描述
fetchIngService 檢查 localStorage serviceTimestamp，now - timestamp >= 600000ms（10 分鐘）則不恢復。getChatInfo 回傳需二次 JSON.parse（data.data → objectRes.Data）。
影響
超過 10 分鐘的對話不會自動恢復。
PM 對策
用戶回報「之前的對話不見了」可能是此機制。
