1 客服進入流程 L1

玩家點擊客服圖示後的完整判斷鏈：線道數量 → csType → iframe 模式

主流程：openCustomService()
點擊客服圖示 → POST /config/getCsLineConfig → 取得線道設定 → VIP 等級篩選
只有 1 條線道 → 直接開啟
多條線道 → BottomSheet 選擇 → 選擇後開啟
csType 分流
csType = 1（官方客服） → IframeContainer → kfUrl + token/sign
csType = 2（第三方客服） → IframeCommon → 第三方網址
幣商跳轉（imDealer） → 隨機取 csType=1 線道 → 直接開啟
iframe 狀態機
close（未開啟） ↔ large（全版 100dvh） ↔ small（半版 325px 浮動） ↔ hidden（暫離保持對話）
logic

VIP 等級篩選

每條線道有 vipBegin ~ vipEnd，只有玩家 VIP 符合範圍才顯示

logic

sType 切換確認

切換不同 sType（line/live/agent）且有進行中對話 → 彈出確認視窗（「回到對話」or「開啟新對話」）

warning

全局 persistent

客服 iframe 掛在 app layout 層，不隨頁面切換卸載；IframeCommon 和 IframeContainer 同時存在 DOM

components/customService/
api

sign 加密

官方客服 URL params 含 uid/uname/token/sign（MD5），kefuSign 是 runtime config 環境變數
