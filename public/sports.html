規則單注流程

點賠率 → 購物車展開 → 輸入金額 → 確認 → 送出 → 顯示「等待結果」→ 成功 / 失敗彈窗。

UI購物車畫面 程式層

從上到下：場館名稱 + 餘額、選中的盤口 + @賠率、金額輸入框、可贏金額提示、送出按鈕。

版面、欄位順序、文案全部寫死。

規則下注金額上下限 設定層

每筆下注的最低 / 最高金額是場館 + VIP 等級雙軸決定的，後端回傳。快捷金額按鈕（100 / 500 / 1000…）也是後台配的。

業主想「調整最低下注額」要去後台改 VIP x 場館配置，不用發版。但修改公式（例：加一個新的計算維度）要發版。

規則賠率即時變動 場館限制

玩家在購物車裡猶豫的時候，後端會推最新賠率過來。賠率變了會提示，玩家可以接受新賠率、拒絕、或設定「自動接受更好的賠率」。

賠率本身動不了（場館給的）。但「變動後的處理規則」我們可以改（接受 / 拒絕 / 彈窗）——那是前端寫死的。

規則重複下注保護 程式層

同一盤口 + 同樣金額 + 短時間內重複點 → 彈窗確認「你是不是按到兩次？」玩家可以勾「不再提示」。

冷卻時間、觸發條件都是寫死的。

規則餘額不足 程式層

金額 > 餘額 → 彈窗提示「餘額不足」，按鈕直接導去充值頁。

測試帳號會顯示「此為測試帳號，無法操作」而不是導去充值。

規則五種下注來源 程式層

購物車可以從這五個地方打開：

  • 首頁簡易盤直接點賠率
  • 推單（主播推的單被玩家點進去）
  • 跟單（複製其他玩家的注單）
  • 追注（上一注輸了追加）
  • 首存包賠（預先幫玩家擺好）
動購物車的需求 每條來源的 flow 都要跑一次。別只驗「從賽事列表點進去」這一條。
