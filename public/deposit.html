PM 提醒：通道的開關、維護、匯率都是第三方或後台控制的。業主問「為什麼某個通道不見了？」第一步查雲平台通道開關，不是報 Bug。另外通道列表的推薦/熱門標籤也是後台設的，不用發版。
存款主頁（通道列表）

UI通道列表 設定層

推薦標籤（suggestType=1）和熱門標籤（isHot）由業主後台設定。推薦不足 2 個時自動從熱門通道補。

通道排序、推薦/熱門標籤都是後台配的，不用發版。

規則進行中訂單 程式層

有進行中訂單時，通道列表被隱藏，改顯示訂單卡片。不能同時有兩張存款單。

訂單每 15 秒輪詢一次，間隔不可調。

規則充值優惠 設定層

通道有 discount 時顯示優惠百分比。實際優惠 = min(金額×%、單次上限、每週上限) 三重上限。

單次/每週上限由後台控。USDT 匯率通道級優先全域。

UI首存引導 設定層

首存未完成的玩家顯示首存優惠 Banner，引導至包賠活動。

USDT 充值（完整展開）
訂單進行中（微信）

規則通道類型分支 程式層

不同 channelType 走不同 UI：Merge 合併充值、Expand 步驟式、代客充值跳客服頁。

  • 代客充值：iframe 嵌入客服系統，有「官方認證」標籤
  • USDT 快捷：先上傳憑證再下單，流程反轉
新增通道類型要全面回測 UI 分支，47 種 payType 已膨脹。

規則重拉單機制 場館限制

下單失敗 10 秒內自動重試。超過 10 秒才顯示錯誤。

重試邏輯寫死，PM 動不了。

sequenceDiagram
    participant U as 使用者
    participant FE as 前端
    participant BE as 後端
    participant TP as 三方支付

    U->>FE: 選擇通道+輸入金額
    FE->>FE: 驗證表單
    FE->>BE: POST /quickpay/payment/rechargepayurl
    Note over FE,BE: payGold 單位為分
    BE->>TP: 轉發至支付通道
    TP-->>BE: 回傳結果
    alt code=1 成功
        BE-->>FE: 訂單(orderId+payUrl)
        alt payUrl.type=1
            FE->>U: 開啟支付頁面
        else payUrl.type=4
            FE->>U: 顯示收款卡資訊
        end
    else code!=1 且10秒內
        BE-->>FE: 失敗
        FE->>FE: counter++ 自動重試
        FE->>BE: 再次 POST
    else 超過10秒
        BE-->>FE: 失敗
        FE->>U: Toast 錯誤
    end
  • 重拉單機制 10 秒內自動重試，PM 動不了這個邏輯
  • payUrl.type=4 是 card2card，收款方資訊由後端動態派發

3.6 場館轉帳（單筆 / 一鍵歸集 / 禮金轉入）
sequenceDiagram
    participant U as 使用者
    participant FE as 前端
    participant BE as 後端
    participant V as 場館系統

    U->>FE: 進入轉帳頁
    FE->>BE: POST /wallet/gameListBalance
    BE->>V: 逐場館查餘額
    V-->>BE: 各場館餘額
    BE-->>FE: gameBalanceList[]
    FE->>U: 顯示場館列表+餘額

    alt 單筆轉帳
        U->>FE: 選場館+輸入金額
        FE->>FE: 檢查 activityLock / maintainLock
        alt 場館鎖定
            FE->>U: 禁止轉帳提示
        else 正常
            FE->>BE: POST /wallet/gameAmountTransferV2
            Note over FE,BE: params: gameOutId, gameInId, gold(分)
            BE->>V: 執行轉帳
            V-->>BE: 結果+遊戲URL
            BE-->>FE: outGold, inGold, url
            FE->>U: 更新餘額顯示
        end
    else 一鍵歸集
        U->>FE: 點「一鍵回收」
        FE->>FE: 冷卻檢查(10秒倒數)
        alt 冷卻中
            FE->>U: 等待提示
        else 可執行
            FE->>BE: POST /wallet/collectAll
            BE->>V: 逐場館歸集
            V-->>BE: 歸集結果
            BE-->>FE: 更新 totalBalance, playerGold
            FE->>U: 餘額刷新
        end
    else 邀請禮金轉入
        U->>FE: 點轉入中心錢包
        FE->>BE: POST /wallet/invitedGoldTransfer
        Note over FE,BE: gold 需乘100(元轉分)
        BE-->>FE: 轉帳結果
        FE->>U: 更新餘額
    end
  • 金額單位：API 以「分」計算，前端顯示轉元
  • activityLock 活動鎖定金額不可轉出；maintainLock 場館維護中不可操作
  • 一鍵歸集有 10 秒冷卻，防連點
