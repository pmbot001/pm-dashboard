12 跨服務限制

場館限制

第三方場館控制，無法由我方調整
場館維護狀態由場館方 API 控制
描述
serverAvailable 由場館 API 回傳，我方無法覆蓋。
影響
場館臨時維護時，玩家立即看到維護遮罩。
PM 對策
無法提前預知場館維護。
場館 API 回應格式不統一
描述
各場館的 gameToken、遊戲進入 URL 格式不同，前端需逐一適配。
影響
新接場館需要開發適配層。
PM 對策
新場館接入排期需預留適配時間。
體育場館只在特定業主顯示
描述
lobbyType=1 (Sport) 只在 TB/YXRG 業主顯示。
影響
其他業主看不到體育分類。
PM 對策
確認業主是否需要體育場館。

模組耦合

三套皮膚各自維護
描述
水果2.0、YXB、YXR 各有獨立 Vue 檔案，功能改動需同步三處。
影響
開發工時 ×3，漏改風險高。
PM 對策
需求單需標注影響哪些皮膚。
單錢包/多錢包模式切換邏輯散落
描述
GOR-4837 整頓單多錢包共用邏輯已暫停，目前分散處理。
影響
轉帳失敗但遊戲已開啟的邊界狀況。
PM 對策
錢包相關改動需全面回歸測試。
AI 推薦依賴後端演算法
描述
AI 精選推薦由後端計算近7日行為，前端只負責展示。
影響
推薦結果不可由 PM 直接控制。
PM 對策
僅能透過後台開關決定是否顯示。

技術限制

LobbyType 枚舉 9 種，持續膨脹
描述
每次新場館類型需擴充枚舉，影響所有皮膚的分流邏輯。
影響
前端改動面大，回歸測試多。
PM 對策
新場館進入方式需先確認是否要新 lobbyType。
外開遊戲 channelId 傳遞需手動帶
描述
window.open 不走 router-guard，channelId 需手動加入 URL。
影響
遺漏 channelId 會導致場館 API 異常。
PM 對策
QA 需測試外開遊戲的 channelId 傳遞。
H5 跳轉外鏈需提示彈窗
描述
QIP-13214 規定 H5 跳轉外鏈場館需先跳提示彈窗。
影響
增加一步操作，影響轉化率。
PM 對策
QIP 規定，不可繞過。

業務限制

場館排序由雲平台設定
描述
classifySort 由雲平台 > 創建渠道設定，前端依此排列。
影響
不同渠道可有不同場館排序。
PM 對策
新渠道上線需確認場館排序設定。
收藏功能僅限子遊戲
描述
場館級別無收藏功能，只有自研大廳的子遊戲支援收藏。
影響
玩家無法收藏常用場館。
PM 對策
如需場館收藏需新開需求。
場館黑名單已整合至 gameList API
描述
GOR-5579 將場館黑名單 API 整合至 /user/gameList。
影響
黑名單場館不會出現在列表中。
PM 對策
黑名單由後台統一管理。
