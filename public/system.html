14 跨服務限制

技術限制

系統層的技術約束，影響所有模組
Token 72 小時有效期不可調
描述
Token 維持 72 小時，延長機制持續遊玩不踢下線。設備號變動 → Token 失效（跟 IP 無關）。
影響
玩家換手機必定需要重新登入，新設備還需簡訊驗證。
PM 對策
客服需要有「新設備無法登入」的 FAQ。
AES 密碼加密 key 寫死在前���
描述
密碼加密用固定 32 位 key + 全零 IV，localStorage 和 API 傳輸共用。
影響
安全性依賴 HTTPS 傳輸層，key 洩漏可解密本地密碼。
PM 對策
暫無替代方案，已知風險��
初始化 watch 觸發 25+ 非同步請求
描述
isLogin + isInitSuccess 變化時觸發大量模組初始化 API（活動/VIP/直播/任務/場館/錢包等）。
���響
首屏載入效能瓶頸，低速網路體感明顯。
PM 對策
新增模組初始化需評估效能影響。
全站 Timeout 統一 20 秒
描述
全站通則規定 Timeout 20 秒（原 10 秒）。沙巴體育超 20 秒顯示彈窗。
影響
低速環境可能大量 timeout，影響體驗。
PM 對策
不可單獨調整，是全站規則。

模組耦合

系統模組與其他模組的依賴關係
channelId 全程跟著 URL query
描述
channelId 必須在每次路由跳轉中從 from 帶到 to，router-guard 統一處理。
影響
任何新路由如果 bypass router-guard（如 window.open），channelId 會遺失。
PM 對策
外開連結需手動帶 channelId query。
登入成功後觸發全模組初始化
描述
登入 → setToken → watch 觸發 → 25+ 模組初始化。包括活動、VIP、直播、場館、錢包等。
影響
登入後有短暫空白期，各模組資料陸續載入。
PM 對策
登入後的跳轉頁面需容忍資料尚未就緒。
維護模式從 baseConfig 取得，非即時推送
描述
維護狀態寫在 baseConfig/v2 的 noticeList 裡，初始化時取得。非 WebSocket 即時推送。
影響
已開啟的玩家需等下次 config 拉取才跳維護頁。
PM 對策
維護公告需提前發布。
路由守衛清空直播購物車
描述
每次路由切換都會 useLiveCartsStore.$reset()。
影響
從直播間跳到其他頁再回來，購物車會被清空。
PM 對策
已知行為，直播購物車不跨頁保存。
客服 iframe 全局持久不卸載
描述
客服 iframe 掛在 app layout 層，不隨頁面切換卸載。排隊視窗固定在右側。
影響
全螢幕橫屏或三方遊戲內隱藏整個客服 UI。
PM 對策
橫屏遊戲中無法使用客服。

設定層限制

需要後台設定配合的系統約束
登入/註冊模式由 baseConfig 控制
描述
accountLoginEnabled / phoneNumberLoginEnabled / accountRegisterEnabled / phoneNumberRegisterEnabled 四個開關。
影響
開關組合錯誤（如全關）有防呆但體驗差。

新業主上線前確認登入/註冊開關設定。
IP 黑名單由雲平台 channelIpBlackList 控制
描述
每次初始化並行拉取黑名單，命中即封鎖。
影響
封鎖是即時的（下次載入生效），解封需後台操作。
PM 對策
誤封需聯繫後台管理員。

影響
不同渠道的玩家登入後看到的首頁不同。

新渠道上線需確認 pushType 設定。

全站通則：重新整理後需停留在原有頁面。Token 失效時刷新回到 pushType 頁。
影響
重整場景的跳轉由 useSystemCheck 處理，不走 router-guard 的 needLogin 攔截。

這是刻意的雙軌設計，避免雙重觸發。

描述
H5/APP：判斷 iOS scheme 或 Android Package name → 有符合開啟 APP；未符合開啟「跳轉參數」。
影響
PC 無 APP，統一用跳轉參數。
PM 對策
跳轉連結需確認三端（PC/H5/APP）行為。
