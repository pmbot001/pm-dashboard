單筆限額

每個通道有 representative.minGold (最低) 和 representative.maxGold (最高)。表單提交前驗證金額是否在此範圍內。

UI 顯示格式：單筆限額：¥min - ¥max

每日次數限制

gamePayTypes.dailyMax = 每日最大次數，remainingTimes = 今日剩餘次數。
當 remainingTimes == 0 && dailyMax >= 0 時，該支付類型被封鎖。

UI 顯示：X / Y 次 (剩餘/每日上限)

共用組限額

多個支付類型可共享存款次數。由 gamePayTypes.sharedGroupId 控制：

6 優惠系統 (Discount)
存款加送機制，有 5 種顯示狀態
5.1.1.9.1 未達最低優惠金額
5.1.1.9.3 已達本週上限
優惠

優惠欄位 (per channel)

isOpen = 1 開啟 | discountPer = 優惠百分比 | minAmount = 最低觸發金額(分) | maxDiscount = 日上限(分) | maxDiscountWeek = 週上限(分)

優惠

5 種顯示狀態

1. 隱藏 — isOpen!=1 或 discountPer≤0
2. 已達週限 — curDiscountWeek ≥ maxDiscountWeek
3. 已達日限 — curDiscount ≥ maxDiscount
4. 未達最低 — amount*100 < minAmount
5. 可領取 — 充值加送 X%

優惠

紅利計算

bonus = min(discountPer * amount / 100, maxDiscount - curDiscount, maxDiscountWeek - curDiscountWeek)

快捷金額按鈕上對達標金額顯示優惠百分比標籤。

必填 | 數字 ≥ 1 | 在 limitRange.min ~ limitRange.max 範圍內

姓名驗證 (29 種類型需要)

Regex: /^(?=.*[a-zA-Z\u4e00-\u9fa5])[a-zA-Z\u4e00-\u9fa5\s]{2,}$/

手機綁定攔截

進入存款頁首先檢查 isBindPhone。未綁定時：封鎖所有支付類型，顯示 BindPhone 元件引導綁定。

風控

實名認證攔截 (DD錢包)

僅 Type 40 (DDP) 要求。檢查 userStore.info?.realName，缺少則導向 /mine/mine-center/real-name。

風控

通道維護攔截

switchStatus 值：1=正常, 2=關閉(後端不回傳), 3=維護中。
狀態 3 或通道列表為空：顯示 MaintainPay 元件，封鎖所有操作。

風控

失敗次數導客服

追蹤三種失敗：GetOrderFailed(3) / SystemAutoCancel(12) / UserManualCancel(13)。
當計數器 ≥ 閾值，引導用戶聯繫客服而非繼續存款。

流程

USDT_DIY (Type 26) — 快捷支付

Expand 通道 → 用戶選擇幣種地址 → 輸入 CNY 金額 → 自動換算 USDT (cryptoAmount/100) → 提交 → 顯示付款資訊 + 30min 倒計時 → 憑證自動上傳

不可取消。付款頁顯示：$realBuyNum USDT ≈ ¥goldAmount

適用類型：code 11, 13, 26, 33, 35, 45

匯率來源：gamePayTypes?.cryptoRate 優先，fallback usdtRate

公式：USDT = CNY / (rate / 100)

倒計時

預設 30 分鐘（從 createTime 起算），若後端有 expireTime 則使用後端值。格式 HH:MM:SS。歸零時自動刷新訂單狀態。

取消規則

不可取消：Type 10 (快捷), Type 26 (USDT快捷)
每日取消上限：cancelDayCnt < maxCancelPerDay
達上限提示：已達本日取消限制
取消需選擇原因（後端回傳 cancelReason 選項列表）
